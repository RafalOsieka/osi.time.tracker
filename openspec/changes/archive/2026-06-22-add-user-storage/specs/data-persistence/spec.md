## ADDED Requirements

### Requirement: REQ-NFR-015 Domain table conventions (UUIDv7 + camelCase)
All domain tables SHALL use a primary key declared as `uuid PRIMARY KEY DEFAULT uuidv7()`, relying on the PostgreSQL ≥ 18 native `uuidv7()` generator so values are time-ordered and produced correctly for every insert (including raw SQL and seed scripts). Column names SHALL use camelCase. The `users` table establishes this convention, and future domain tables SHALL follow it.


#### Scenario: Domain table uses UUIDv7 primary key
- **WHEN** a domain table is created
- **THEN** its primary key SHALL be `uuid` with a `DEFAULT uuidv7()` clause, and inserts that omit the id SHALL receive a time-ordered UUIDv7 generated by PostgreSQL

#### Scenario: Domain table uses camelCase columns
- **WHEN** a domain table defines columns
- **THEN** the column identifiers SHALL be written in camelCase (e.g. `passwordHash`, `displayName`, `createdAt`)

#### Scenario: Requires PostgreSQL 18 or newer
- **WHEN** the migration is applied against a PostgreSQL server older than 18
- **THEN** the `uuidv7()` default SHALL fail, signalling the documented minimum-version deployment constraint
